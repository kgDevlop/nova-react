# Spreads

`SpreadsEditor` in [`src/apps/spreads.jsx`](../src/apps/spreads.jsx).

## Storage shape

The doc's serialised content is a flat object keyed by A1 references:

```js
cells = {
  "A1": { raw: "=B2+1", fmt: "$" },
  "B2": { raw: "12" },
  ...
}
```

`raw` is exactly what the user typed. Display values are computed on read —
nothing about formatting or formulas is materialised on write.

## Display pipeline

1. `_display(key)` looks up `cells[key]`.
2. If `raw` starts with `=`, it is fed to `_evalFormula(raw, cells)` from
   [`shared/formulas.js`](../src/shared/formulas.js).
3. Numeric results are formatted by `cell.fmt`:
   - `pct` → percent with 1 dp
   - `$`   → currency
   - else, plain number toString.

## Visible viewport

`VISIBLE_COLS = 16`, `VISIBLE_ROWS = 60`. The grid is fixed-size — Spreads
does *not* virtualise rows. Adding more rows means raising the constant.

`COLS = ["A".."P"]` is generated by `_colLetter(i)` from `formulas.js`,
which already supports the `Z → AA` rollover for any future expansion.

## Spread tabs

```js
const [spreads, setSpreads] = useState(["Spread 1", "Spread 2", "Spread 3"]);
```

Spread tabs are bookkeeping-only today — switching `activeSpread` does **not**
yet partition the `cells` map.

## Conditional highlighting (planned)

The original spec called for conditional highlighting based on cell value
or formula result. Tracked in [[issues]]. The fastest implementation path is
a per-cell `cell.cf` rule list with `_evalFormula`-style predicates.
